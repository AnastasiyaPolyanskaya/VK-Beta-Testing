# Проект №3: IVI — iOS Functional Testing (Referral System Bug)

## Описание
Проект содержит тест-кейс для проверки критического дефекта в системе реферальных начислений мобильного приложения **IVI (iOS)**.  
Обнаружена уязвимость, позволяющая пользователям получать бонусные монеты без выполнения требуемых действий по приглашению друзей.

Проект оформлен в формате QA-документации с детальным описанием бага, шагами воспроизведения и ожидаемым поведением системы.

## Цель тестирования
- Выявить и задокументировать критическую уязвимость в системе начисления реферальных бонусов
- Проверить корректность логики начисления монет за выполнение задания "Пригласить друга"
- Верифицировать отсутствие возможности обхода бизнес-логики приложения

## Тип тестирования
- Security testing (проверка уязвимостей)
- Functional testing
- Negative testing

## Окружение
- ОС: iOS 18.2.6  
- Устройство: iPhone 14 Pro Max  
- Платформа: VK / VK Mini Apps  
- Приложение: LoFi Кот Барсик  
- Тип тестирования: Exploratory (исследовательское)  
- Применимость: Mobile (iOS)

## Тестовые данные
- Активный пользовательский аккаунт
- Задание "Пригласить друга +10 монет" в системе бонусов

## Объект тестирования
- Система бонусных монет приложения
- Модуль реферальных программ ("Пригласить друга")
- Логика начисления вознаграждений
- Интерфейс управления монетами (кнопка "плюс" в левом верхнем углу)

## Шаги воспроизведения дефекта
1. Нажать на кнопку "плюс" (монета в левом верхнем углу интерфейса)
2. Выбрать задание «Пригласить друга +10 монет»
3. Закрыть окно для приглашения друзей (без выполнения приглашения)
4. Проверить баланс монет

## Фактический результат
Монеты за задание «Пригласить друга» начисляются пользователю без фактического приглашения друга и отправки реферальной ссылки.

## Ожидаемый результат
Монеты за задание «Пригласить друга» должны начисляться **только после** успешного приглашения и регистрации нового пользователя по реферальной ссылке.

## Серьезность дефекта
**Critical** — уязвимость позволяет пользователям неограниченно получать бонусные монеты, обходя бизнес-логику приложения, что может привести к:
- Финансовым потерям компании
- Нарушению экономики внутренней валюты приложения
- Несправедливому преимуществу пользователей, эксплуатирующих баг

## Приоритет исправления
**High** — требуется немедленное исправление ввиду критичности уязвимости.


## Статус тестирования
**Выполнено:** 1 тест-кейс  
**Результат:** Failed (обнаружена критическая уязвимость)  
**Дефект:** Зарегистрирован в баг-трекинговой системе
