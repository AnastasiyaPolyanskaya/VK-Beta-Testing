# Тест-кейсы - Проект №3  
IVI: Security Testing (Referral System Vulnerability)

## Описание
В данном разделе представлен тест-кейс для проверки критической уязвимости в системе реферальных начислений мобильного приложения **IVI (iOS)**.

Тест-кейс направлен на проверку:
- корректности логики начисления бонусных монет
- безопасности системы вознаграждений
- отсутствия возможности обхода бизнес-логики приложения

## Тип тестирования
- Security testing (тестирование безопасности)
- Functional testing (функциональное тестирование)
- Negative testing (негативное тестирование)
- Exploit testing (тестирование на уязвимости)

## Объект тестирования
- Мобильное приложение IVI (iOS)
- Раздел: Система бонусных монет
- Модуль: Реферальная программа "Пригласить друга"
- Элемент интерфейса: Кнопка "плюс" (монета в левом верхнем углу)

## Основные проверяемые сценарии
- Доступность задания "Пригласить друга +10 монет" в системе бонусов
- Корректность начисления монет при выполнении условий задания
- Отсутствие возможности получения награды без выполнения обязательных действий
- Безопасность процесса начисления бонусов

## Критическая уязвимость
Обнаружена возможность получения бонусных монет без фактического выполнения условия задания "Пригласить друга".

## Шаги воспроизведения уязвимости
1. Нажать на кнопку "плюс" (изображение монеты в левом верхнем углу интерфейса)
2. В открывшемся меню выбрать задание «Пригласить друга +10 монет»
3. Закрыть всплывающее окно для приглашения друзей (без фактического приглашения)
4. Проверить баланс бонусных монет

## Фактический результат
Система начисляет 10 бонусных монет за задание «Пригласить друга» без выполнения пользователем условия приглашения.

## Ожидаемый результат
Система должна начислять бонусные монеты только после:
1. Фактического приглашения друга через реферальную ссылку
2. Успешной регистрации приглашенного пользователя
3. Подтверждения выполнения условия задания системой

## Серьезность дефекта
**Critical** - критическая уязвимость системы безопасности

## Приоритет исправления
**High** - высокий приоритет, требуется немедленное исправление
