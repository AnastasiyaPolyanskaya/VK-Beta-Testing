# Проект №3: VK - Бета-тестирование. Приложение "Кворик"
## Баг с навигацией после создания квора

## Описание проекта
Во время тестирования приложения "Кворик" в бета-версии VK обнаружен дефект навигации: при создании нового квора и последующем нажатии кнопки "Назад" пользователь не возвращается в ожидаемый раздел, а попадает обратно в окно создания квора.

## 1. Описание дефекта
После успешного создания нового квора в приложении "Кворик" нажатие системной кнопки "Назад" приводит пользователя не в панель управления кворами (ожидаемое поведение), а снова открывает форму создания квора. Это нарушает логику навигации и создает путаницу для пользователя.

## 2. Шаги воспроизведения
1. Открыть панель управления кворами в приложении "Кворик"
2. Нажать кнопку "Создать новый квор"
3. Ввести название квора в соответствующее поле
4. Нажать кнопку "Создать квор"
5. После успешного создания нажать кнопку "Назад" (системную или в интерфейсе приложения)

## 3. Фактический результат
Пользователь возвращается в окно создания квора (форма с полем для ввода названия), а не в панель управления кворами.

## 4. Ожидаемый результат
После создания квора и нажатия кнопки "Назад" пользователь должен вернуться в панель управления кворами, где отображаются все созданные кворы.
